◆ destroyWordsArr()

void destroyWordsArr ( char **  toDestroy,
int  logicalSize 
)

désalloue tout les tableaux alloués dynamiquement pour ToDestroy

toDestroy représente un tableau de chaine de caractère, chaque éléments ayant été alloué dynamiquement. Afin de désallouer proprement un tableau généré par loadWords, veuillez utiliser cette méthode. Désalloue non seulement le tableau toDestroy lui même, mais aussi les tableaux chaînes de char qu'il contient.

Paramètres
toDestroytableau de string à désallouer
logicalSizenombre de string contenu dans toDestroy, et donc à désallouer

◆ emptyStream()

int emptyStream ( FILE *  stream,
int  nbCharDumped 
)

vide un flux d'un certain nombre de charactère

ATTENTION! si le buffer spécifié est stdin et est vide, attendra la saisie d'un charactère sur cet input Si nbCharDumped est négatif, vide tout les charactères. Sinon vide le nombre spécifié par nbCharDumped

Paramètres
streamflux dont il faut vider tout les charactères. Le flux n'est ni ouvert ni fermé, charge à l'appelant de le faire
nbCharDumpednombre de charactère à jeter. Si ce nombre est négatif, vide tout le flux. Sinon vide le nombre spécifié de charactère.
Renvoie
0 si tout s'est bien déroulé
valeur non nulle si quelque chose s'est mal passé (se référer au code de retour de ferror())

Fonction permettant de lancer un jeu du pendu.

Utiliser cette fonction permet d'avoir tout le comportement du pendu

Renvoie
-1 si la récupération d'un mot pour le jeu c'est mal passé, ou que l'alphabet n'a pu etre alloué
0 si le jeu a été perdu, donc que le mot n'a pu etre deviné 1 si le mot a été deviné

Récupère d'un fichier au formatage particulier des mots, et les stockes dans myTab.

Stocke dans storingTab les chaînes de caractères se trouvant dans le fichier indiqué par pathToFile
ATTENTION! le chemin du fichier est relatif à la position de l'utilisateur, et non de l'executable lui même

Paramètres
pathToFilechemin vers le fichier dont il faut récupérer les données
storingTabstocke l'adresse du tableau de string créé (passage par adresse)
Renvoie
-2 si le fichier n'a pu etre ouvert en mode lecture ou mal lu
-1 si une allocation dynamique n'a pu etre effectur
sinon, retourne le nombre de chaine de caractère récupérés
